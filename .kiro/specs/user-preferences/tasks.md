# Implementation Plan

- [x] 1. ユーザープリファレンス管理フック
- [x] 1.1 カレンダーデフォルトビューを管理するカスタムフックを作成する
  - 既存の `useLocalStorage` フックを利用し、カレンダーデフォルトビュー（月/週/日）の読み書きを型安全に一元管理する
  - デフォルト値として月ビューを設定し、不正な値が保存されていた場合もフォールバックする
  - localStorage への書き込み失敗時にも state として値を保持し、セッション中は動作を継続する
  - 将来の設定項目追加に対応可能な拡張性を意識した設計にする
  - _Requirements: 3.3, 5.1, 5.2, 5.3, 5.5_
  - _Contracts: useUserPreferences Service_

- [ ] 2. テーマ設定パネルコンポーネント
- [ ] 2.1 (P) テーマ選択パネルを作成する
  - 「表示設定」セクション内にテーマ選択UIを構築し、ライト・ダーク・システムの3つの選択肢を提供する
  - next-themes の `useTheme()` を利用して現在のテーマ取得と変更を行い、DashboardHeader の ThemeSwitcher と同一のテーマ状態を共有する
  - 現在適用中のテーマを視覚的にハイライト表示する
  - テーマ変更が即座にUI全体に反映されることを確認する
  - ハイドレーションミスマッチを回避するため、マウント状態を管理してSSR時はフォールバックUIを表示する
  - Storybook ストーリー（CSF3形式）を作成し、各テーマ選択状態のバリアントを定義する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5_

- [ ] 3. カレンダーデフォルトビュー設定パネルコンポーネント
- [ ] 3.1 カレンダーデフォルトビュー選択パネルを作成する
  - 「表示設定」セクション内にカレンダーデフォルトビュー選択UIを構築し、月ビュー・週ビュー・日ビューの3つの選択肢を提供する
  - `useUserPreferences` フックを利用してデフォルトビュー値の読み取りと更新を行う
  - 選択変更時に永続化を実行し、保存成功のフィードバック（チェックアイコン + 保存完了メッセージ）を表示する
  - 保存失敗時にエラーメッセージを表示する
  - フィードバックは操作後3秒以内に表示し、その後自動消去する
  - Storybook ストーリー（CSF3形式）を作成し、通常状態・保存成功・エラー状態のバリアントを定義する
  - _Requirements: 3.1, 3.2, 3.3, 5.4, 6.1, 6.2, 6.3_

- [ ] 4. ユーザーメニューコンポーネント
- [ ] 4.1 (P) DashboardHeader 内のユーザーアバタードロップダウンメニューを作成する
  - ユーザーアバターをトリガーとする DropdownMenu を構築する
  - メニューアイテムとしてプロフィール、設定、ログアウトの3つを提供する
  - プロフィールは `/dashboard/user` へ、設定は `/dashboard/user/settings` へ遷移させる
  - ログアウト機能は既存の LogoutButton のロジックを DropdownMenuItem として統合する
  - Storybook ストーリー（CSF3形式）を作成し、メニュー開閉状態のバリアントを定義する
  - _Requirements: 4.2, 4.3_

- [ ] 5. 設定ページのルーティングとレイアウト
- [ ] 5.1 設定ページの Server Component とレイアウトを作成する
  - `/dashboard/user/settings` ルートに認証チェック付きの Server Component ページを作成する
  - 未認証ユーザーがアクセスした場合はログインページにリダイレクトする
  - 既存のユーザープロフィールページと同一の認証パターンを踏襲する
  - テスト容易性のためレイアウトを表示専用の Client Component として分離する
  - DashboardHeader を含み、ダッシュボードへ戻るナビゲーションリンクを提供する
  - 設定カテゴリごとにセクション分けされたレイアウトで、テーマ設定パネルとカレンダーデフォルトビュー設定パネルを配置する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_

- [ ] 6. 既存コンポーネントへの導線追加と統合
- [ ] 6.1 DashboardHeader にユーザーメニューを統合する
  - 現在のアバターリンクと LogoutButton をユーザーメニューコンポーネントに置き換える
  - ThemeSwitcher はヘッダー内に独立して維持し、クイックアクセスを保つ
  - DashboardHeader は Server Component のまま維持し、UserMenu を Client Component として分離する
  - 既存のテスト・ストーリーを更新して統合後の表示と遷移を確認する
  - _Requirements: 4.2_

- [ ] 6.2 (P) UserProfileCard に設定ページへのリンクを追加する
  - プロフィールカード内に設定ページ (`/dashboard/user/settings`) へのアイコンリンクを追加する
  - 既存のプロフィール表示ロジックは変更しない
  - 既存のテスト・ストーリーを更新してリンクの表示と遷移を確認する
  - _Requirements: 4.1, 4.3_

- [ ] 7. カレンダー初期表示へのデフォルトビュー反映
- [ ] 7.1 カレンダーURL同期フックにデフォルトビュー注入オプションを追加する
  - `useCalendarUrlSync` にオプション引数 `defaultViewMode` を追加し、URL パラメータ未指定時のフォールバック値を外部から注入可能にする
  - URL に `?view=` パラメータが存在する場合はそちらを優先し、ディープリンク対応を維持する
  - 引数なしの既存呼び出しに対する後方互換性を保つ
  - _Requirements: 3.4, 3.5_

- [ ] 7.2 カレンダーコンテナでユーザー設定のデフォルトビューを適用する
  - カレンダーコンテナ側で `useUserPreferences` からデフォルトビュー値を取得し、URL同期フックに渡す
  - デフォルトビュー設定が存在しない場合は月ビューをフォールバックとして使用する
  - _Requirements: 3.4, 3.5, 5.2_

- [ ] 8. 単体テストとコンポーネントテスト
- [ ] 8.1 (P) ユーザープリファレンスフックの単体テストを作成する
  - デフォルト値（月ビュー）の返却を検証する
  - 値の更新と localStorage への永続化を検証する
  - 不正な値が保存されている場合のフォールバック動作を検証する
  - _Requirements: 5.1, 5.2, 5.5_

- [ ] 8.2 (P) テーマ設定パネルのコンポーネントテストを作成する
  - 3つのテーマオプション（ライト/ダーク/システム）の表示を検証する
  - テーマ選択時に setTheme が正しく呼び出されることを検証する
  - 現在のテーマのハイライト表示を検証する

- [ ] 8.3 (P) カレンダーデフォルトビュー設定パネルのコンポーネントテストを作成する
  - 3つのビューオプション（月/週/日）の表示を検証する
  - 選択変更時の永続化呼び出しを検証する
  - 保存成功フィードバックの表示と自動消去を検証する
  - 保存失敗時のエラー表示を検証する
  - _Requirements: 3.1, 3.2, 3.3, 5.4, 6.1, 6.2, 6.3_

- [ ] 8.4 (P) ユーザーメニューのコンポーネントテストを作成する

  - 各リンクの遷移先が正しいことを検証する

- [ ]*8.6 (P) 設定ページレイアウトの描画テストを作成する
  - ページタイトル・セクション見出し・戻るリンクの描画を検証する
  - テーマ設定パネルとカレンダーデフォルトビュー設定パネルが表示されることを検証する
