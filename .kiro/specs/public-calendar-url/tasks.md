# Implementation Plan

- [x] 1. DBスキーマ拡張とRLSポリシー追加
- [x] 1.1 guildsテーブルに公開カレンダー用カラムを追加するマイグレーションを作成する
  - guildsテーブルに公開フラグ（デフォルトfalse）と公開スラッグ（NULL許容、UNIQUE制約付き）カラムを追加する
  - 公開スラッグに部分インデックスを作成し、NULLでないレコードのみを対象にする
  - 既存レコードは公開フラグfalse・スラッグNULLで初期化される（デフォルト値による自動適用）
  - _Requirements: 1.4_

- [x] 1.2 匿名ユーザー向けRLSポリシーを追加するマイグレーションを作成する
  - anonロールに対して、公開フラグがオンかつ未削除のギルドの読み取りを許可するSELECTポリシーを追加する
  - anonロールに対して、公開ギルドに紐づくイベントの読み取りを許可するSELECTポリシーを追加する
  - anonロールに対して、公開ギルドに紐づくイベントシリーズの読み取りを許可するSELECTポリシーを追加する
  - INSERT/UPDATE/DELETEは一切許可しないことを確認する
  - _Requirements: 3.3, 3.4_

- [x] 2. 型定義の拡張とサービス層の実装
- [x] 2.1 Guild型とデータ変換関数に公開カレンダー用フィールドを追加する
  - GuildRow型にDB側のスネークケースフィールド（公開フラグ、公開スラッグ）を追加する
  - Guild型にキャメルケースのドメインフィールド（isPublic、publicSlug）を追加する
  - toGuild変換関数にスネークケースからキャメルケースへのマッピングを追加する
  - _Requirements: 1.4_

- [x] 2.2 公開カレンダー専用のサービスを実装する
  - 公開スラッグからギルド情報を取得する機能を実装する（非公開・不存在の場合はエラーを返す）
  - 公開ギルドのイベントを期間指定で取得する機能を実装する（繰り返しイベントのオカレンス展開を含む）
  - 公開カレンダーの有効化（スラッグ生成）・無効化・スラッグ再生成の機能を実装する
  - スラッグ生成にはcrypto.randomUUIDベースの12文字hexを使用し、UNIQUE制約違反時は最大3回リトライする
  - ギルドの公開設定（有効/無効、現在のスラッグ）を取得する機能を実装する
  - 無効化時はスラッグを保持し、再有効化時に再利用可能とする
  - 返却データからチャンネル情報・通知設定・メンバーリストなど内部情報を除外する
  - Result型パターンに従い、成功/失敗の判別共用体を返す
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 3.3, 3.4_

- [x] 2.3 (P) 公開カレンダーサービスの単体テストを実装する
  - スラッグからの公開ギルド取得、非公開ギルドの拒否、存在しないギルドの拒否をテストする
  - 公開カレンダー有効化時のスラッグ生成、UNIQUE制約違反時のリトライ動作をテストする
  - イベント取得、繰り返しイベントのオカレンス展開、チャンネル情報の除外をテストする
  - 無効化時のスラッグ保持と再有効化動作をテストする
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 3.3, 3.4_

- [ ] 3. ミドルウェアとServer Actionsの実装
- [ ] 3.1 ミドルウェアの公開ルート判定にカレンダーパスを追加する
  - ミドルウェアの公開ルートリストにカレンダー用パスプレフィックスを追加し、認証チェックをバイパスする
  - 既存の公開ルート判定ロジック（プレフィックスマッチ）に従って追加する
  - _Requirements: 2.5_

- [ ] 3.2 公開カレンダー設定を操作するServer Actionsを実装する
  - 公開カレンダーの有効化/無効化を切り替えるServer Actionを実装する（認証+MANAGE_GUILD権限チェック付き）
  - 公開スラッグを再生成するServer Actionを実装する（認証+MANAGE_GUILD権限チェック付き）
  - resolveServerAuthによる認証チェックとcanManageGuildによる権限検証を行う
  - 操作成功後にrevalidatePathで管理画面のキャッシュを無効化する
  - 既存のServer Actionsパターン（Result型、classifySupabaseError）に従う
  - _Requirements: 1.1, 1.2, 1.3_

- [ ] 4. 公開カレンダー閲覧ページの実装
- [ ] 4.1 公開カレンダーページのServer Componentを実装する
  - URLパラメータのスラッグからギルドを解決し、公開状態を検証するServer Componentを作成する
  - 非公開または存在しないギルドの場合はnotFound()で404ページを返す（403ではなく404で存在を隠蔽）
  - 初期表示月のイベントデータをSSRで取得し、クライアントコンポーネントに渡す
  - ギルド名をページヘッダーに表示する
  - _Requirements: 2.1, 2.2, 2.5, 3.1, 3.2_

- [ ] 4.2 公開カレンダーページのOGP/メタタグを実装する
  - generateMetadataでOpen Graphメタタグ（og:title、og:description、og:image、og:url）を動的に生成する
  - og:titleにギルド名を含める
  - og:descriptionにギルドの直近イベント情報（最大3件のイベント名）を含める
  - Twitterカード用メタタグ（twitter:card、twitter:title、twitter:description）を設定する
  - OGP画像は初期リリースではデフォルトブランド画像を使用する
  - 既存のgenerateMetadataパターン（docs/[slug]）に準拠する
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5_

- [ ] 4.3 公開カレンダー表示用のクライアントコンテナコンポーネントを実装する
  - 読み取り専用モードでカレンダーグリッドとツールバーを組み合わせた表示コンテナを作成する
  - 月ナビゲーション（前月・次月・今日）のインタラクションを管理する
  - ナビゲーション時にクライアントサイドでanonロールを使ってイベントデータを再フェッチする
  - AbortControllerで前回のフェッチリクエストをキャンセルする
  - イベント追加ボタン・設定ボタン・ドラッグ操作・編集/削除/RSVPのUIを一切表示しない
  - イベントクリック時は読み取り専用の詳細表示（タイトル、日時、説明、場所）のみ提供する
  - _Requirements: 2.1, 2.2, 2.3, 2.4_

- [ ] 5. 管理画面の公開URL設定UIの実装
- [ ] 5.1 ギルド設定画面に公開カレンダーURL設定セクションを追加する
  - 既存のギルド設定フォーム内に公開カレンダーURL設定セクションを追加する
  - 公開カレンダーの有効・無効を切り替えるトグルUIを提供する
  - 公開設定がオンの場合は現在の公開URLとコピーボタンを表示する
  - 公開設定がオフの場合はURL生成を促すUIを表示する
  - コピーボタンはclipboard APIを使用し、コピー成功時にフィードバックを表示する
  - 公開URLのフォーマットはベースURL + カレンダーパス + スラッグとする
  - useTransitionでオプティミスティック更新を行う
  - _Requirements: 1.5, 5.1, 5.2, 5.3, 5.4_

- [ ] 5.2 公開設定の無効化・スラッグ再生成時の確認ダイアログを実装する
  - 公開設定を無効化する際に、無効化の影響（既存の共有URLが無効になること）を説明する確認ダイアログを表示する
  - スラッグを再生成する際に、旧URLが無効になることを説明する確認ダイアログを表示する
  - スラッグ再生成ボタンは公開設定がオンの場合のみ表示する
  - _Requirements: 1.3, 5.5_

- [ ] 6. 統合テストとE2Eテスト
- [ ] 6.1 ミドルウェアとRLSポリシーの統合テストを実装する
  - カレンダーパスが公開ルートとして認識され、認証チェックがバイパスされることを検証する
  - anonロールで公開ギルドのイベントのみ読み取れること、非公開ギルドのデータが返却されないことを検証する
  - anonロールでINSERT/UPDATE/DELETEが拒否されることを検証する
  - _Requirements: 2.5, 3.3, 3.4_

- [ ]* 6.2 (P) 公開カレンダーページとOGPの動作確認テストを実装する
  - 有効な公開URLにアクセスしてカレンダーが正常に表示されることを検証する
  - 非公開ギルドのURLにアクセスして404が表示されることを検証する
  - 存在しないスラッグにアクセスして404が表示されることを検証する
  - OGPメタタグにギルド名とイベント概要が含まれていることを検証する
  - _Requirements: 2.1, 2.2, 3.1, 3.2, 4.1, 4.2, 4.3_

- [ ]* 6.3 (P) 管理画面の公開URL設定UIの動作確認テストを実装する
  - 公開URLの生成・コピー・無効化が正常に動作することを検証する
  - 無効化時に確認ダイアログが表示されることを検証する
  - スラッグ再生成後に新しいURLが表示されることを検証する
  - _Requirements: 1.1, 1.2, 1.3, 1.5, 5.1, 5.2, 5.3, 5.4, 5.5_
