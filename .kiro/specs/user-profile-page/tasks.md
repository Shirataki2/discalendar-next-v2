# Implementation Plan

- [x] 1. UIコンポーネントを実装する
- [x] 1.1 (P) プロフィールカードコンポーネントを実装する
  - ユーザーのDiscordアバター画像を大きめの円形で表示する
  - アバターが取得できない場合は表示名の頭文字をイニシャルとしてフォールバック表示する
  - ユーザーの表示名を表示し、表示名が未設定の場合はメールアドレスにフォールバックする
  - メールアドレスをサブテキストとして表示する
  - shadcn/ui の Card と Avatar コンポーネントを活用して統一感のあるUIにする
  - _Requirements: 2.1, 2.2, 2.3, 2.4_

- [x] 1.2 (P) 参加ギルド一覧コンポーネントを実装する
  - ユーザーが参加しているギルドをアイコンと名前付きのリストで表示する
  - 各ギルド項目をクリックすると、そのギルドが選択された状態でダッシュボードに遷移する
  - ギルドが0件の場合は「参加しているギルドがありません」というメッセージを表示する
  - hover時に視覚的なフィードバックを提供する
  - _Requirements: 3.1, 3.2, 3.3_

- [x] 2. ユーザーページを作成してデータ取得とレイアウトを統合する
  - Supabase AuthからユーザーのDiscordメタデータ（アバター、表示名、メール）を取得する
  - Guild Serviceからユーザーが参加しているギルド一覧を取得する
  - プロフィールカード、ギルド一覧、ログアウトボタンを配置してページを構成する
  - ログアウトボタンは既存のコンポーネントを再利用し、セッション破棄とリダイレクトを行う
  - モバイル幅ではシングルカラム、デスクトップ幅では適切な余白とカード配置のレスポンシブレイアウトにする
  - すべてのインタラクティブ要素がタッチ操作でアクセス可能にする
  - ギルド取得に失敗した場合もプロフィール表示は継続し、ギルドセクションにエラーメッセージを表示する
  - _Requirements: 1.1, 1.2, 4.1, 4.2, 4.3, 5.1, 5.2, 5.3_

- [x] 3. (P) ダッシュボードヘッダーにユーザーページへのナビゲーションリンクを追加する
  - ダッシュボードヘッダーのアバター部分をクリック可能にし、ユーザーページへのリンクにする
  - hover時の視覚的フィードバックを追加する
  - 既存のアバター表示ロジックは変更しない
  - _Requirements: 1.3_

- [ ] 4. Storybookストーリーと単体テストを作成する
- [ ] 4.1 (P) プロフィールカードのストーリーとテストを作成する
  - 全データあり（Default）、アバターなし（NoAvatar）、表示名なし（NoFullName）のストーリーを作成する
  - アバター画像の正しい表示を検証するテストを作成する
  - アバター未設定時のイニシャルフォールバックを検証するテストを作成する
  - 表示名とメールアドレスの正しい表示を検証するテストを作成する
  - _Requirements: 2.1, 2.2, 2.3, 2.4_

- [ ] 4.2 (P) ギルド一覧のストーリーとテストを作成する
  - 複数ギルドあり（WithGuilds）、0件（EmptyGuilds）、1件（SingleGuild）のストーリーを作成する
  - ギルドリストの正しいレンダリングを検証するテストを作成する
  - 0件時のメッセージ表示を検証するテストを作成する
  - 各ギルドのリンク先URLが正しいことを検証するテストを作成する
  - _Requirements: 3.1, 3.2, 3.3_
