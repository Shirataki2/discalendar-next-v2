# Implementation Plan

- [ ] 1. データベーススキーマの設計と作成
- [x] 1.1 繰り返しイベントシリーズ用のテーブルを作成する
  - `event_series` テーブルをマイグレーションファイルで作成し、ギルドID・イベント名・RRULE文字列・開始日時・期間・除外日リストなどのカラムを定義する
  - RLSポリシーを設定し、認証済みユーザーがCRUD操作を行えるようにする
  - ギルドIDと開始日時にインデックスを作成する
  - `updated_at` 自動更新トリガーを設定する
  - _Requirements: 8.1, 8.2_

- [x] 1.2 既存イベントテーブルに繰り返し関連カラムを追加する
  - `events` テーブルにシリーズ参照カラムと元日付カラムをNULL許容で追加する
  - シリーズ削除時の外部キー動作を設定する（孤立した例外レコードは単発イベントとして残る）
  - シリーズIDにパーシャルインデックスを作成する
  - 既存データに影響がないことを確認する
  - _Requirements: 8.2, 9.2_

- [ ] 2. RRULEユーティリティの実装
- [x] 2.1 (P) `rrule` パッケージの導入とRRULE文字列生成関数を実装する
  - `rrule` npmパッケージをインストールする
  - 頻度（毎日・毎週・毎月・毎年）、間隔、曜日指定、月次モード（日付指定・n番目曜日指定）、終了条件（無期限・回数・日付）からRFC 5545準拠のRRULE文字列を生成する純粋関数を作成する
  - すべての入力パターンに対する単体テストを作成する
  - _Requirements: 1.2, 2.4, 3.2, 3.3, 3.4_
  - _Contracts: rrule-utils buildRruleString_

- [x] 2.2 (P) オカレンス展開関数を実装する
  - RRULE文字列と開始日時から、指定した日付範囲内のオカレンスのみを展開する関数を作成する
  - EXDATE（除外日）を考慮してオカレンスをフィルタリングする
  - 不正なRRULE文字列のパースエラーをハンドリングし、空の展開結果を返しつつログに記録する
  - 範囲内展開と大量オカレンス時のパフォーマンスをテストで検証する
  - _Requirements: 4.1, 4.5, 8.3, 8.4_
  - _Contracts: rrule-utils expandOccurrences_

- [x] 2.3 (P) RRULE要約テキスト変換とバリデーション関数を実装する
  - RRULE文字列を人間が読めるテキスト（例: 「毎週火曜日」「2週間ごと」）に変換する関数を作成する
  - RRULE文字列の妥当性を検証し、有効性と具体的なエラー理由を返す関数を作成する
  - 各パターンの変換結果とバリデーション結果の単体テストを作成する
  - _Requirements: 4.3, 8.4_
  - _Contracts: rrule-utils toSummaryText, validateRrule_

- [ ] 3. EventServiceへの繰り返しイベント機能追加
- [x] 3.1 シリーズ作成機能を実装する
  - イベントシリーズをRRULEルールとともにデータベースに保存するメソッドを追加する
  - Result型パターンに従い、成功時にシリーズレコード、失敗時にエラー情報を返す
  - 入力データのバリデーション（ギルドID必須、RRULE文字列の妥当性検証）を行う
  - タスク1のマイグレーションが適用済みであることが前提
  - _Requirements: 1.3, 8.1, 8.5_
  - _Contracts: EventService createRecurringSeries_

- [x] 3.2 単発イベントと繰り返しイベントの統合取得機能を実装する
  - 指定した日付範囲に対して、単発イベントと繰り返しイベントのオカレンスを統合して取得するメソッドを追加する
  - 各シリーズのRRULEをrrule-utilsで展開し、EXDATEを除外した上でオカレンスを生成する
  - 例外レコード（個別編集済みのオカレンス）を展開結果に反映する
  - 繰り返しフラグ・シリーズID・ルール要約テキスト・元日付を含む拡張されたイベント型で返す
  - 既存の単発イベント取得動作に変更がないことを確認するテストを作成する
  - _Requirements: 1.4, 4.1, 4.4, 8.3, 9.1, 9.3_
  - _Contracts: EventService fetchEventsWithSeries_

- [x] 3.3 単一オカレンスの編集・削除機能を実装する
  - 「この予定のみ」の編集時に、該当オカレンスの変更内容を例外レコードとしてデータベースに保存するメソッドを追加する
  - 「この予定のみ」の削除時に、該当日をシリーズの除外日リストに追加するメソッドを追加する
  - 例外レコードはシリーズIDと元の日付で紐付ける
  - 他のオカレンスに影響がないことをテストで検証する
  - _Requirements: 5.2, 5.4, 8.2_
  - _Contracts: EventService updateOccurrence, deleteOccurrence_

- [x] 3.4 シリーズ全体の編集・削除機能を実装する
  - シリーズのマスター情報（タイトル・時刻・説明等）を一括更新するメソッドを追加する
  - RRULEルール自体の変更に対応し、オカレンス展開結果を再計算する
  - 例外リセットオプションに対応する（全体編集時に個別例外をリセットするかどうか）
  - シリーズ全体と関連する例外データを完全に削除するメソッドを追加する
  - _Requirements: 6.1, 6.2, 6.3, 8.5_
  - _Contracts: EventService updateSeries, deleteSeries_

- [x] 3.5 シリーズ分割と切り詰め機能を実装する
  - 「これ以降の予定」の編集時に、元のシリーズを選択日の前日で終了させ、新しいルールのシリーズを選択日から作成するメソッドを追加する
  - 「これ以降の予定」の削除時に、元のシリーズのRRULEに終了日パラメータを設定して選択日以降のオカレンスを生成しないようにするメソッドを追加する
  - 分割時のエラーハンドリング（中間エラー時に元シリーズの状態を復元する補償処理）を実装する
  - 分割後の各シリーズが独立して編集・削除可能であることをテストで検証する
  - _Requirements: 7.1, 7.2, 7.3, 8.5_
  - _Contracts: EventService splitSeries, truncateSeries_

- [ ] 4. Server Actionsの拡張
- [x] 4.1 繰り返しイベント作成アクションを実装する
  - 既存の権限チェックパターンを再利用し、繰り返しイベントシリーズを作成するServer Actionを追加する
  - ギルドへのアクセス権限を検証した上で、EventServiceのシリーズ作成メソッドに委譲する
  - 繰り返し設定を無効のまま保存した場合は既存の単発イベント作成動作を維持する
  - _Requirements: 1.3, 1.5_

- [ ] 4.2 オカレンス編集・削除アクションを実装する
  - 編集スコープ（この予定のみ / すべての予定 / これ以降の予定）に応じて適切なEventServiceメソッドを呼び分けるServer Actionを追加する
  - 「この予定のみ」は例外レコード作成またはEXDATE追加、「すべての予定」はシリーズ全体更新または削除、「これ以降の予定」はシリーズ分割または切り詰めに委譲する
  - すべてのアクションで権限チェックを実行する
  - _Requirements: 5.2, 5.4, 6.1, 6.2, 6.3, 7.1, 7.2_

- [ ] 5. 繰り返し設定フォームUIの実装
- [ ] 5.1 繰り返し設定のフォーム状態管理フックを実装する
  - 繰り返しの有効/無効、頻度、間隔、曜日、月次モード、終了条件のフォーム状態を管理するカスタムフックを作成する
  - バリデーション機能を実装する（間隔は1以上、回数は1以上、終了日はイベント開始日以降、週次の場合は曜日1つ以上選択）
  - 現在のフォーム状態からRRULE文字列を生成する機能を提供する
  - 既存のイベントフォームフックと並行使用できるようにする
  - _Requirements: 1.1, 2.1, 2.2, 2.3, 3.1, 3.5, 3.6_
  - _Contracts: useRecurrenceForm_

- [ ] 5.2 繰り返し設定入力コンポーネントを実装する
  - イベント作成フォーム内で繰り返しの有効/無効を切り替えるトグルを追加する
  - 頻度選択（毎日・毎週・毎月・毎年）のドロップダウンを実装する
  - 「毎週」選択時に繰り返す曜日を複数選択できるUIを実装する
  - 「毎月」選択時に日付指定またはn番目の曜日指定を選択できるUIを実装する
  - 間隔入力（例: 2週間ごと、3ヶ月ごと）を実装する
  - 終了条件選択（無期限・回数指定・日付指定）のUIを実装する
  - Storybookストーリーを作成する
  - _Requirements: 1.1, 2.1, 2.2, 2.3, 3.1_
  - _Contracts: RecurrenceSettingsControl_

- [ ] 6. 編集・削除スコープ選択UIの実装
- [ ] 6.1 (P) 編集スコープ選択ダイアログを実装する
  - 繰り返しイベントの編集・削除時に「この予定のみ」「すべての予定」「これ以降の予定」の3択を表示する確認ダイアログを作成する
  - 削除時には確認メッセージを表示する
  - 「すべての予定」編集時に個別例外をリセットするかどうかの確認チェックボックスを表示する
  - 既存の確認ダイアログと同様のUIパターンを踏襲する
  - Storybookストーリーを作成する
  - _Requirements: 5.1, 5.3, 6.4_
  - _Contracts: EditScopeDialog_

- [ ] 7.1 繰り返しイベントのカレンダー表示を統合する
  - カレンダーのデータ取得を、単発イベントと繰り返しオカレンスの統合取得に切り替える
  - 表示範囲変更時に該当範囲内のオカレンスのみを動的に展開して表示する
  - 繰り返しイベントのオカレンスに繰り返しアイコンインジケーターを表示する
  - 単発イベントと繰り返しイベントのオカレンスを同一カレンダー上に混在表示する

- [ ] 7.2 イベントプレビューに繰り返し情報を表示する
  - イベントプレビューポップオーバーに繰り返しルールの概要テキスト（例: 「毎週火曜日」）を表示する
  - 繰り返しイベントの場合、編集・削除ボタンクリック時にスコープ選択ダイアログを経由させる
  - 既存のプレビュー・編集ダイアログのUI構造と整合する形式を維持する

  - フォーム送信時に繰り返し設定の有無に応じて適切なServer Action（単発 or シリーズ）を呼び分ける
  - 繰り返しイベントの編集時にスコープに応じたフォーム初期値を設定する

  - スコープに応じて適切なServer Actionを呼び出し、カレンダー表示を更新する
  - エラー発生時のトースト通知と既存のエラーコード（シリーズ未検出・RRULEパースエラー）に対応する
